- 三个层次相对独立，修改位置不重叠

```cpp
if (ch >= 'a' && ch <= 'z') {
  // 处理标识符和关键字
  k = 0;
  do {
    if (k < al) {
      a[k] = ch;
      k++;
    }
    getchdo;
  } while (ch >= 'a' && ch <= 'z' || ch >= '0' && ch <= '9');
  // ...
} else if (ch >= '0' && ch <= '9') {
  // 处理数字
  // ...
} else {
  // 处理特殊符号
  if (ch == ':') { /*检测赋值符号*/ }
  else if (ch == '<') { /*检测小于或小于等于符号*/ }
  else if (ch == '>') { /*检测大于或大于等于符号*/ }
  else { sym = ssym[ch]; }
}
```

### 2. 关键字识别使用折半查找

| 构造NFA识别token | 手写状态机逻辑 | ✅ |
| 转换为DFA优化 | 条件分支判断 | ✅ |
| 使用lex/flex工具 | 手写getsym() | ✅ |

## 为什么不用显式的NFA/DFA？

1. **PL0语言简单**：词法规则少，手写更直观
2. **教学目的**：代码清晰，便于理解编译原理
3. **效率足够**：直接匹配对小语言性能很好
4. **可维护性**：手写代码更容易调试和修改

## 如果要使用NFA/DFA

通常需要：
1. 用正则表达式定义词法规则
2. 使用lex/flex等工具生成词法分析器
3. 工具会自动构建NFA→DFA→优化代码

但PL0选择手写实现，更适合教学和理解编译器工作原理。

## 递归下降的体现

### 1. 每个非终结符对应一个函数
PL0的语法结构通过函数映射：

| 语句 | `statement()` | [第655行](file:///d:\课件\编译原理\编译原理实验\compiler-expt\PL0\PL0.cpp#L655) |
| 表达式 | `expression()` | [第820行](file:///d:\课件\编译原理\编译原理实验\compiler-expt\PL0\PL0.cpp#L820) |
| 项 | `term()` | [第860行](file:///d:\课件\编译原理\编译原理实验\compiler-expt\PL0\PL0.cpp#L860) |
| 因子 | `factor()` | [第900行](file:///d:\课件\编译原理\编译原理实验\compiler-expt\PL0\PL0.cpp#L900) |

### 2. 函数间相互调用形成递归

**表达式 → 项 → 因子 → 表达式（递归）**
```cpp
// expression() 调用 term()
int expression(bool *fsys, int *ptx, int lev) {
  // ...
  termdo(nxtlev, ptx, lev);  // 调用term()
  // ...
}

// term() 调用 factor()
int term(bool *fsys, int *ptx, int lev) {
  // ...
  factordo(nxtlev, ptx, lev);  // 调用factor()
  // ...
}

// factor() 可能调用 expression()（括号内表达式）
int factor(bool *fsys, int *ptx, int lev) {
  if (sym == lparen) {
    getsymdo;
    expressiondo(nxtlev, ptx, lev);  // 递归调用expression()
    // ...
  }
}
```

**程序块 → 语句 → 程序块（递归）**
```cpp
// block() 调用 statement()
int block(int lev, int tx, bool *fsys) {
  // ...
  statementdo(nxtlev, &tx, lev);  // 调用statement()
  // ...
}

// statement() 处理过程调用时可能回到block()
if (sym == callsym) {
  // ...
  gendo(cal, lev - table[i].level, table[i].adr);  // 调用过程
}
```

### 3. 递归下降的语法结构

```
程序 → block()
  ↓
声明部分 → constdeclaration() / vardeclaration()
  ↓
语句部分 → statement()
  ↓
表达式 → expression()
  ↓
项 → term()
  ↓
因子 → factor()
  ↓
(可能回到expression()形成递归)
```

## 递归下降的优势

1. **直观清晰**：代码结构与语法规则一一对应
2. **易于理解**：每个函数处理一个语法结构
3. **便于扩展**：新增语法规则只需添加新函数
4. **错误处理简单**：可以在每个函数中检测语法错误

## 与其他方法对比

| 方法 | PL0使用 | 特点 |
|------|---------|------|
| 递归下降 | ✅ | 手写，直观，适合教学 |
| LL(1)分析表 | ❌ | 需要构造分析表 |
| LR分析 | ❌ | 自底向上，更复杂 |
| YACC/Bison | ❌ | 自动生成分析器 |

PL0选择递归下降是因为语言简单，手写实现既清晰又高效。

## 四、语法结构映射

```
程序 → block()
  ├── 声明部分 → constdeclaration() / vardeclaration() / block()
  └── 语句部分 → statement()
        ├── 赋值语句 → expression()
        ├── if语句 → condition() + statement()
        ├── while语句 → condition() + statement()
        ├── begin语句 → statement() + ... + statement()
        └── call语句

表达式 → expression()
  └── 项 → term()
        └── 因子 → factor()
              ├── 标识符
              ├── 数字
              └── (表达式)

条件 → condition()
  ├── odd表达式
  └── 表达式 关系运算符 表达式
```

## 五、关键字映射

每个关键字在语法分析中对应特定的处理逻辑：

| 关键字 | 处理位置 | 语法角色 |
|-------|---------|---------|
| `const` | `block()` | 常量声明开始 |
| `var` | `block()` | 变量声明开始 |
| `procedure` | `block()` | 过程声明开始 |
| `begin` | `statement()` | 复合语句开始 |
| `end` | `statement()` | 复合语句结束 |
| `if` | `statement()` | 条件语句开始 |
| `then` | `statement()` | if语句分隔符 |
| `while` | `statement()` | 循环语句开始 |
| `do` | `statement()` | while语句分隔符 |
| `call` | `statement()` | 过程调用 |
| `read` | `statement()` | 输入语句 |
| `write` | `statement()` | 输出语句 |
| `odd` | `condition()` | 奇数判断 |

这种设计是典型的**递归下降分析法**，每个非终结符对应一个函数，函数间相互调用形成递归结构。
