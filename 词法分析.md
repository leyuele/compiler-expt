# PL/0 编译器代码分析

这是一个经典的 **PL/0 编译器**实现，包含完整的词法分析、语法分析、代码生成和虚拟机解释执行功能。

---

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    PL/0 编译器流程                          │
├─────────────────────────────────────────────────────────────┤
│  源程序(.PL0) → 词法分析 → 语法分析 → 中间代码生成 → 解释执行  │
│                     ↓              ↓              ↓         │
│                   getsym()      block()        interpret() │
└─────────────────────────────────────────────────────────────┘
```

## 文件功能说明

### 1. PL0.h - 头文件

定义了编译器的核心数据结构和常量：

| 类别 | 定义内容 |
|------|----------|
| **常量** | 关键字数量(norw=13)、名字表容量(txmax=100)、地址上界(amax=2047)、最大嵌套层数(levmax=3)、最大代码数(cxmax=200) |
| **符号枚举** | `symbol` - 32种词法符号（关键字、运算符、分隔符） |
| **对象类型** | `object` - constant/variable/procedur |
| **指令类型** | `fct` - lit/opr/lod/sto/cal/inte/jmp/jpc |
| **指令结构** | `instruction` - {f:操作码, l:层次, a:地址/值} |
| **名字表项** | `tablestruct` - {name, kind, val, level, adr, size} |

### 2. PL0.cpp - 主实现文件

实现了编译器的五个核心模块：

#### (1) 初始化模块 (`init()`)
- 初始化字符符号表 `ssym[]`（单字符符号映射）
- 初始化关键字表 `word[]` 和 `wsym[]`（按字母排序，支持二分查找）
- 初始化指令助记符 `mnemonic[]`
- 初始化符号集合（声明开始集、语句开始集、因子开始集）

#### (2) 词法分析器 (`getsym()`)
- **核心功能**：从源程序读取字符，识别并返回词法符号
- **处理逻辑**：
  - 跳过空白字符（空格、换行、制表符）
  - 识别标识符/关键字：通过二分查找匹配关键字表
  - 识别数字：累加计算数值
  - 识别复合符号：`:=`、`<=`、`>=`
  - 识别单字符符号：通过 `ssym[]` 查表

#### (3) 语法分析与代码生成

采用**递归下降分析法**，对应 PL/0 语法规则：

| 函数 | 对应语法规则 | 功能 |
|------|-------------|------|
| `block()` | 分程序 | 处理声明（常量/变量/过程）+ 语句序列 |
| `statement()` | 语句 | 处理赋值、read、write、call、if、while、复合语句 |
| `condition()` | 条件 | 处理 odd 和关系表达式 |
| `expression()` | 表达式 | 处理加减运算 |
| `term()` | 项 | 处理乘除运算 |
| `factor()` | 因子 | 处理标识符、数字、括号表达式 |

**代码生成示例**：
```c
// 赋值语句 a := 88;
gen(lit, 0, 88);   // 将常量88推入栈顶
gen(sto, 0, 3);    // 栈顶值存入变量a（相对地址3）

// 表达式 a + b * (3 + b)
gen(lod, 0, 3);    // 取a的值
gen(lod, 0, 4);    // 取b的值
gen(lit, 0, 3);    // 常量3
gen(lod, 0, 4);    // 取b的值
gen(opr, 0, 2);    // 3 + b
gen(opr, 0, 4);    // b * (3 + b)
gen(opr, 0, 2);    // a + b * (3 + b)
```

#### (4) 符号表管理
- `enter()` - 向符号表添加常量/变量/过程
- `position()` - 在符号表中查找标识符位置

#### (5) 虚拟机解释器 (`interpret()`)
- **栈式虚拟机**，使用栈 `s[]` 执行中间代码
- **核心寄存器**：
  - `p` - 程序计数器（指令指针）
  - `b` - 基址寄存器（当前过程数据区基地址）
  - `t` - 栈顶指针
- **指令执行**：

| 指令 | 操作码 | 功能说明 |
|------|--------|----------|
| `lit` | 0 | 将常量值a推入栈顶 |
| `opr` | 1 | 执行算术/逻辑运算（a=0退出过程，a=2加法，a=6取模等） |
| `lod` | 2 | 加载变量值到栈顶 |
| `sto` | 3 | 存储栈顶值到变量 |
| `cal` | 4 | 调用过程 |
| `inte` | 5 | 分配数据区空间 |
| `jmp` | 6 | 无条件跳转 |
| `jpc` | 7 | 条件跳转 |

### 3. E01.PL0 - 示例程序

```pl0
var a,b,c;           // 变量声明
begin
  a := 88;           // 赋值
  read(b);           // 输入
  c := a + b * (3 + b);  // 表达式计算
  write(c)           // 输出
end.                 // 程序结束
```

## 执行流程

1. **输入阶段**：用户输入PL/0源文件名
2. **配置阶段**：选择是否输出虚拟机代码和名字表
3. **编译阶段**：
   - 词法分析：`getsym()` 逐个读取符号
   - 语法分析：`block()` 递归解析分程序结构
   - 代码生成：`gen()` 生成三地址码存入 `code[]`
4. **解释阶段**：`interpret()` 执行生成的虚拟机代码
5. **输出阶段**：结果写入 `fa2.tmp`，中间代码写入 `fa.tmp`，名字表写入 `fas.tmp`

---

## 技术特点

- **单遍编译**：词法分析、语法分析、代码生成同时进行
- **递归下降**：直观映射语法规则到递归函数
- **栈式虚拟机**：简单高效，支持过程嵌套
- **符号表**：支持按层次查找，实现作用域管理
- **错误处理**：检测多种语法错误并报告位置

这是编译原理课程中经典的教学实现，完整展示了编译器从源码到执行的全过程。

```cpp
if (ch == '<') /*检测小于、小于等于或不等于符号*/
{
  getchdo;
  if (ch == '=') {
    sym = leq;      // <=
    getchdo;
  } else if (ch == '>') {
    sym = neq;      // <> 不等于
    getchdo;
  } else {
    sym = lss;      // <
  }
}
```
